## Overview

**Sentinella** is a self-hosted web application designed for LAN environments where you need to monitor the web without relying on paid external services.

It combines:

- up-to-date web search through **SearXNG**
- local **Markdown** digests generated by **Ollama**
- **scheduled watchlists** with cron and URL deduplication
- **multi-user** access with JWT and `admin` / `user` roles

The goal is simple: turn recurring searches into an operational workflow that is searchable, traceable, and manageable from a web interface.

## Highlights

| Area | What it provides |
| --- | --- |
| Ask | one-shot search with AI digest and cited sources |
| Watchlist | recurring monitoring powered by APScheduler |
| Run history | execution and result tracking |
| Multi-user | login, roles, and user/admin visibility rules |
| Local-first | full stack runnable entirely with Docker Compose |
| No API key | web search runs on SearXNG, not external providers |

## Architecture

```text
Host :8001
   |
 nginx
   |-- /        -> frontend React/Vite
   `-- /api/*   -> FastAPI
                    |-- PostgreSQL
                    |-- SearXNG + Redis
                    |-- Ollama
                    `-- Worker APScheduler
```

Main components:

- **Frontend**: React 18 + Vite
- **API**: FastAPI with JWT authentication, RBAC, and rate limiting
- **Worker**: dedicated scheduler for personal and global watchlists
- **Search**: SearXNG with JSON output
- **AI digest**: Ollama with a local model
- **Database**: PostgreSQL 16

## Quick Start

```bash
cp .env.example .env
# set at least JWT_SECRET
docker compose up --build
```

Initial access:

- UI: `http://localhost:8001`
- Health: `http://localhost:8001/api/health`
- Default login: `admin` / `admin123`

Operational notes:

- the configured Ollama model is downloaded on first startup
- change the admin password immediately
- for a full reset: `docker compose down -v`

## Workflow

1. The user runs a query from **Ask** or creates a **watchlist**.
2. Sentinella queries **SearXNG** and collects relevant URLs.
3. Content is fetched, extracted, and normalized.
4. **Ollama** generates a Markdown digest based only on the collected sources.
5. The result is stored as a **run** and can be reviewed in the history.

## Why Sentinella

- keeps data under control inside the local infrastructure
- reduces manual work for repetitive searches
- makes queries, sources, and digests searchable over time
- clearly separates ad hoc search, periodic monitoring, and user administration
