# Data Model

The database is PostgreSQL 16. Tables are created automatically by SQLAlchemy with `Base.metadata.create_all` when the API starts (no migrations).

## Tables

### `users`

| Column | Type | Notes |
|---------|------|------|
| `id` | INTEGER PK | Auto-increment |
| `username` | VARCHAR UNIQUE | Unique username |
| `password_hash` | VARCHAR | bcrypt hash |
| `role` | VARCHAR | `admin` or `user` |
| `is_active` | BOOLEAN | Default `true` |
| `created_at` | TIMESTAMP | Creation time |

**Bootstrap**: if the table is empty at startup, `admin` / `admin123` is created automatically.

---

### `watchlist`

| Column | Type | Notes |
|---------|------|------|
| `id` | INTEGER PK | — |
| `name` | VARCHAR | Descriptive name |
| `query` | TEXT | Search query |
| `cron` | VARCHAR | Cron expression (for example `"0 8 * * *"`) |
| `enabled` | BOOLEAN | Default `true` |
| `scope` | VARCHAR | `global` or `personal` |
| `owner_user_id` | INTEGER FK `users.id` | NULL for global |
| `recency_days` | INTEGER | Default 7 |
| `max_results` | INTEGER | Default 5 |
| `domains_allow` | JSONB | Whitelisted domain list |
| `domains_block` | JSONB | Blacklisted domain list |
| `created_at` | TIMESTAMP | — |

**Indexes**: `scope`, `owner_user_id`

---

### `runs`

| Column | Type | Notes |
|---------|------|------|
| `id` | INTEGER PK | — |
| `watch_id` | INTEGER FK `watchlist.id` | NULL for runs created by `/api/ask` |
| `user_id` | INTEGER FK `users.id` | NULL for global watch runs |
| `query` | TEXT | Query used |
| `items` | JSONB | Array of `{title, url, snippet, text}` |
| `digest_md` | TEXT | Markdown digest generated by Ollama |
| `created_at` | TIMESTAMP | — |

**Indexes**: `watch_id`, `user_id`

---

### `seen_items`

| Column | Type | Notes |
|---------|------|------|
| `id` | INTEGER PK | — |
| `watch_id` | INTEGER FK `watchlist.id` | — |
| `url_hash` | VARCHAR | SHA256 of the URL |
| `url` | TEXT | Original URL |
| `first_seen` | TIMESTAMP | First time seen |

**UNIQUE index**: `(watch_id, url_hash)` — guarantees per-watch deduplication.

**Purpose**: avoid including URLs already processed in previous runs of the same watch.

---

## Relationships

```
users (1) ──< watchlist (personal, owner_user_id)
users (1) ──< runs (user_id)
watchlist (1) ──< runs (watch_id)
watchlist (1) ──< seen_items (watch_id)
```

## Backup

```bash
# Manual dump
docker compose exec -T postgres pg_dump -U assistant assistant > backup.sql

# Restore
docker compose exec -T postgres psql -U assistant assistant < backup.sql
```

Daily cron example on the host:
```bash
0 2 * * * cd /path/to/sentinella && docker compose exec -T postgres pg_dump -U assistant assistant > /backup/sentinella_$(date +\%F).sql
```
